EDA · Digital Notes
01 · What is EDA?

Exploratory Data Analysis

EDA is the process of understanding and summarizing the main characteristics of a dataset before applying any machine learning model. It helps discover patterns, detect anomalies, and check assumptions.

STEP 01
Profile of Data
  • No. of rows
  • No. of columns
  • Size of data
  • Missing values
  • Data types (numerical / categorical)
  • Duplicate values
STEP 02
Statistical Analysis
  • Mean, Median, Mode
  • Std Dev, Min, Max, Quantiles
  • Frequency of categorical columns
  • Variance of columns
  • Covariance of columns
  • t-test, chi-square, ANOVA, z-test
STEP 03
Graph-Based Analysis
  • Histogram — for numerical columns
  • Bar chart — for categorical columns
  • Box plot — outlier + distribution check
  • Scatter plot — linearity + correlation
  • Heatmap — correlation between variables
  • KDE — probability density of continuous var
  • Pie chart — proportion of categories

03 · Handling Missing Values

Why Handle Missing Values? interview ★

Missing data decreases predictive power. Applying algorithms on missing data introduces bias in parameter estimation — you cannot trust results if missing values are not handled.

Reasons for Missing Values

  • People do not give information (survey skips)
  • Inaccuracy during data collection
  • Data accumulated from various past records
  • Equipment inconsistency leading to faulty measurement

Types of Missing Values interview ★

TypeFull FormMeaningExample
MCAR Missing Completely At Random No pattern — purely random missing Random sensor failure
MAR Missing At Random Missingness depends on other observed variables Income missing more for younger people
NMAR Not Missing At Random Missingness depends on the missing value itself (intentional) High-income earners skip income question

How to Deal with Missing Values

CCA — Complete Case Analysis

  • Simply drop rows with missing values
  • Only valid when data is MCAR
  • Risk: lose too much data

Imputation

Simple (1 column at a time)

  • Numerical: Mean / Median / Random / End of distribution
  • Categorical: Mode / "Missing" as new category

Multivariate (all columns)

  • KNN Imputer
  • Iterative Imputer (MICE)
Rule of thumb: Use Mean when data is normally distributed. Use Median when data has outliers (skewed). Use Mode for categorical data.
04 · Handling Outliers

What are Outliers? interview ★

Outliers are data points that are far separate from the majority of data points. If not handled during EDA, they lead to biased model results and poor business decisions.

Impact on algorithms: Affects Linear Regression, Logistic Regression, AdaBoost, Deep Learning. Tree-based algorithms (Random Forest, XGBoost) are NOT affected by outliers.

How to Detect Outliers

MethodWhen to UseHow
Z-ScoreData is Normally Distributed|Z| > 3 → outlier. Z = (x − μ) / σ
Box Plot (IQR)Data is NOT Normally DistributedBelow Q1 − 1.5×IQR or above Q3 + 1.5×IQR
Percentile-basedGeneral purposeDecide upper/lower tail (e.g. 1%–99%)

How to Deal with Outliers

Trimming

Completely remove the outlier rows

Capping

Fix upper/lower limit (upper tail + lower tail)

Missing Value

Treat outlier as missing → impute

Discretization

Group into bins (mygroup)

IQR = Q3 − Q1  |  Lower = Q1 − 1.5×IQR  |  Upper = Q3 + 1.5×IQR
05 · Feature Scaling

Why Scale? interview ★

If features have varying scales, large-scale features dominate the model. Scaling brings all features into a comparable range, ensuring optimization algorithms converge faster and more stably.

No effect on: Decision Trees, Random Forest, XGBoost — tree-based algorithms are not distance-based, so scaling doesn't affect them.

1. Standardization Z-Score Normalization

x' = (x − mean) / std_dev
  • Mean = 0, Std Dev = 1
  • Range: (−∞, +∞) — no fixed bounds
  • Typical range: (−3, +3)
  • Use when data is normally distributed
  • Works well with Gradient Descent, SVM, Neural Nets
sklearn: StandardScaler()

2. Normalization Min-Max Scaling

x' = (x − min) / (max − min)
  • Min = 0, Max = 1
  • Range: [0, 1] — fixed bounds
  • Use when data has a fixed range
  • Use for CNN image input (0 to 255 → 0 to 1)
  • Sensitive to outliers
sklearn: MinMaxScaler()

Advantages of Scaling

BenefitExplanation
Faster TrainingLeads to faster convergence during optimization
Better GeneralizationAll features contribute equally — no domination
Algorithm StabilityOptimization algorithms work more smoothly
06 · Data Transformation

What is Data Transformation?

Applying mathematical functions to convert the PDF (Probability Density Function) of a column to be closer to a Normal Distribution — leads to better model performance.

Why necessary? Skewed data makes it difficult to identify meaningful patterns. Skewness introduces bias into statistical analysis. Converting to normal helps linear models perform better.

Function Transformations

MethodUsed For
Log (log x)+ve skewed (right skewed) data
Reciprocal (1/x)Convert large → small, small → large
Square (x²)Left skewed data
Square Root (√x)Mild right skew — not used much
CustomDomain-specific transformations

Power Transformations

MethodConstraintUsed For
Box-Coxx > 0 onlyPositive data only
Yeo-JohnsonAll valuesBoth −ve and +ve data
Power transformations work better than function transformations — try both and select whichever gives better results.
07 · Encoding Categorical Features

Why Encode? interview ★

ML algorithms require numerical data. Categorical variables represent qualitative attributes — they must be converted to numerical form through encoding.

1. Label Encoding

  • Assign integer to each category
  • S, M, L, XL, XXL → 1, 2, 3, 4, 5
  • Use for ordinal data (order matters)
  • Example: Rating: 1★, 2★, 3★
⚠️ Can imply false ordering in nominal data

2. One-Hot Encoding

  • Creates new binary column per category
  • Red, Green, Blue → 3 columns (0/1)
  • Use for nominal data (no order)
  • Dummy variable trap — drop one column
Use when categories are few (<10)

3. Binary Encoding

  • Convert to binary representation
  • Fewer columns than one-hot
  • Good for high-cardinality features
  • Example: 100 categories → ~7 binary cols
Use when many unique categories exist

Encoding Summary

EncodingData TypeCardinalityUse When
LabelOrdinalAnyOrder matters (S<M<L<XL)
One-HotNominalLow (<10)No order, few categories
BinaryNominalHighMany unique values
TargetNominalHighReplace category with target mean
08 · Imbalanced Data

What is Class Imbalance? interview ★

Data imbalance occurs when the distribution of classes in a dataset is uneven — one class significantly outnumbers another. ML algorithms become biased towards the majority class.

Why it matters: Model trained on imbalanced data often learns to predict the majority class always → high accuracy but poor recall for minority class. Standard accuracy becomes misleading.

Techniques to Handle Imbalance

UNDERSAMPLING

Reduce majority class representation by removing samples

0: 233017 → 133 1: 347 → 347
  • Reduces by 80%
  • Applied when lots of data
  • Risk: lose useful data
OVERSAMPLING

Increase minority class representation by duplicating samples

0: 233017 → 233017 1: 347 → 33508
  • Increases by 50%+
  • Risk: overfitting duplicate data
SMOTE

Synthetic Minority Oversampling Technique — generates synthetic samples by interpolating between existing minority samples

  • Combination of under + oversampling
  • Based on nearest neighbours
  • Parameter: pct (0–1, 2–100)
  • More diverse than simple duplication

Conclusion

Focus on Precision, Recall, F1 Score — not accuracy. Use domain knowledge to reduce FP or FN. Based on that, select ROC score or probability threshold (e.g. 0.1–0.6). Use SMOTE + oversampling techniques with Ensemble methods (RF, XGBoost) providing class_weight parameters.
09 · Curse of Dimensionality

What is it? interview ★

There is an optimal number of features for best accuracy. If features are increased beyond that, accuracy decreases because data becomes sparse, computational complexity increases, and algorithm efficiency reduces.

The Problem

  • Data sparsity — in high-dim space, points get far from each other
  • Every element or value within dataset, matrix or vector may be zero or empty
  • Algorithms that rely on dense data struggle
  • Performance decreases, computational cost increases
Used for high-dimensional data: Images and Text

Solutions

  • Feature Selection — select relevant subset, remove redundant features
  • Feature Extraction — create new lower-dimensional features

Problems Caused

  • Performance decreases
  • Computational cost increases

Resolution Methods

  • Feature Selection → Forward / Backward
  • Feature Extraction → PCA / LDA
10 · Feature Selection

What is Feature Selection? interview ★

Process of selecting the most important subset of features from a large dataset. Applied on training data only — never on test data (risk of data leakage).

Why Necessary?

  • Cure the Curse of Dimensionality
  • Decrease computational cost
  • Improve interpretability
  • Reduce inference time, improve accuracy
1. FILTER METHOD
  • Duplicate column removal
  • Variance Threshold
  • Correlation
  • ANOVA
  • Chi-Square
  • Mutual Information
2. WRAPPER METHOD
  • Exhaustive Feature Selection
  • Forward Selection
  • Backward Elimination
  • Recursive Feature Elimination (RFE)
3. EMBEDDED METHOD
  • Lasso (L1 Regularization)
  • Ridge (L2 Regularization)
  • Tree-based importance (RF, XGBoost)
  • Elastic Net

Filter Method — Key Techniques

TechniqueWhen to UseHow
Variance ThresholdFeature has near-zero varianceRemove constant feature (variance = 0) or quasi-constant features
CorrelationInput/Input — numericalIf |corr| > 0.7–0.8 between two features → remove one
ANOVAInput numerical → Output categoricalKeep features with strong F1-Y relation
Chi-SquareInput categorical → Output categoricalStudy difference between ideal and real data; compare p-value
Mutual InformationAny → AnyMeasures amount of info shared between feature and target

Wrapper Method — Forward vs Backward

Forward Selection

  • Start with zero features
  • Add one feature at a time
  • At each stage: select feature that improves model most
  • Stop when no improvement
  • Complexity: n(n+1)/2

Backward Elimination

  • Start with ALL features
  • Remove least important one at a time
  • Stop when removing hurts performance
  • Complexity: n(n+1)/2
  • Disadvantage: local selection (may miss best)

PCA — Principal Component Analysis interview ★★

Unsupervised Dimensionality Reduction

Transforms high-dimensional data into lower-dimensional space while retaining most important information and minimizing information loss.

Purpose

  • Dimensionality Reduction — reduce no. of features, retain key info → faster + simpler model
  • Data Visualization — visualize high-dim data in 2D/3D (100D → 3D/2D)
  • Noise Reduction — remove noise, focus on dominant patterns

How PCA Works — Step by Step

Step 1: Center the data — subtract mean of each feature Step 2: Calculate Covariance Matrix (captures variance between each pair of features) Step 3: Compute Eigen Values & Eigen Vectors Eigen Vector = represent PC (direction of max variance) Eigen Value = how much variance each PC explains Step 4: Sort by Variance — arrange Eigen Vectors by Eigen Value (highest to lowest variance) Step 5: Project Data on Principal Components Select subset of Eigen Vectors to reduce dimensionality

Finding Optimal No. of PCs

λ₁ + λ₂ + ... + λₖ ≥ 90% of (λ₁ + λ₂ + ... + λₙ)

Keep components until cumulative variance ≥ 90%

When PCA Does NOT Work

  • Variance of x and y is the same
  • When projection can't differentiate the data
  • Pattern y = x² — losing piece of information

LDA — Linear Discriminant Analysis interview ★

Supervised Classification

Dimensionality reduction technique for supervised classification problems. Projects features from high-dim space to lower dim space (2D to 1D) while maximizing class separation.

Goal

  • Maximize separation between classes
  • Minimize variance within each class
  • Result: lower-dimensional space for classification/visualization

Criteria

  • Class distance between means of 2 classes ↑
  • Class variation within each class ↓

How LDA Works — Step by Step

Step 1: Compute mean of dependent variable Step 2: Calculate covariance matrix of class variable Step 3: Compute Within-class Scatter Matrix (Sw+Se) Step 4: Calculate Between-class Scatter Matrix Step 5: Compute Eigen Vectors & Eigen Values (within class + between scatter) Step 6: Sort values of Eigen Vector by Eigen Value Step 7: Find Eigen Vector corresponding to k Eigen Values Step 8: Obtain LDA — dot product of Eigen Vector with original data

PCA vs LDA

PCALDA
TypeUnsupervisedSupervised
GoalMax varianceMax class separation
Uses labelsNoYes
Best forGeneral reductionClassification + viz

13 · Data Drift Added

What is Data Drift? interview ★★

Data Drift (also called Dataset Shift) occurs when the statistical properties of input data change over time after a model is deployed — causing model performance to degrade even though the model itself hasn't changed.

Core Problem: A model trained on historical data may perform well initially, but as the real-world data distribution shifts over time, the model becomes less accurate. The world changes — the model doesn't.

Simple Analogy

Scenario: You train a spam filter in 2020. By 2024, spammers use new words, new patterns, new tactics. → Your model still looks for old patterns. → Performance drops — this is Data Drift.

Types of Data Drift interview ★★

1. COVARIATE SHIFT

Input distribution changes

P(X) changes but P(Y|X) remains same

  • Most common type
  • Feature values shift
  • Example: model trained on young users, now used by older users — age distribution shifted
2. LABEL / CONCEPT DRIFT

Relationship between X and Y changes

P(Y|X) changes — same features, different meaning

  • Model logic becomes outdated
  • Example: "Fraud" behavior patterns change — old fraud rules no longer apply
  • Customer preferences shift
3. PRIOR PROBABILITY SHIFT

Output distribution changes

P(Y) changes — class proportions shift

  • Class balance shifts over time
  • Example: fraud rate was 1% at training, now 10% in production
  • Seasonal changes in classes

Data Drift vs Model Drift

Data DriftModel Drift
What changesInput/Output data distributionModel performance metrics
CauseReal-world patterns shiftData drift OR model degradation
DetectionStatistical tests on dataMonitor accuracy, F1, AUC
FixRetrain model on new dataRetrain, revalidate, redeploy

How to Detect Data Drift interview ★

Statistical Tests

TestData TypeWhat it Checks
KS TestNumericalKolmogorov-Smirnov: compares two distributions
Chi-SquareCategoricalDistribution shift in categories
PSIAnyPopulation Stability Index — most common in banking
Jensen-ShannonAnyMeasures divergence between two distributions
WassersteinNumericalEarth mover's distance — how much distribution shifts

PSI — Population Stability Index interview ★

PSI = Σ (Actual% − Expected%) × ln(Actual% / Expected%)
PSI ValueInterpretation
< 0.1No significant drift ✅
0.1 – 0.2Moderate drift — monitor closely
> 0.2Significant drift — retrain model ❌
PSI is widely used in credit risk and banking models to monitor production performance.

How to Handle Data Drift

Detection & Monitoring

  • Monitor model performance continuously — accuracy, F1, AUC over time
  • Feature distribution monitoring — check if input feature distributions shift (mean, std dev)
  • Scheduled drift detection — run statistical tests weekly/monthly
  • Shadow mode testing — compare old model vs new model on live data

Fixes

  • Retrain with new data — most common fix; periodically retrain on recent data
  • Online / incremental learning — model updates continuously with new data
  • Sliding window — use only recent N months of data for training
  • Weighted resampling — give more weight to recent data points
  • Ensemble with new model — blend old + new model during transition
  • Feature engineering update — add new features that capture new patterns

Data Drift in the ML Lifecycle

Train Phase: Model learns from historical data ↓ Deploy Phase: Model performs well on similar data ✓ ↓ Monitor Phase: Track performance metrics over time ↓ Drift Detected: PSI > 0.2, accuracy drops, F1 drops ↓ Alert & Retrain: Collect new data → retrain → redeploy ↓ Back to Deploy → cycle continues
Interview tip: Data drift is a key topic in MLOps. Always mention monitoring tools like Evidently AI, WhyLogs, Alibi Detect, or cloud-native solutions like AWS SageMaker Model Monitor, Azure ML Data Drift when discussing production ML systems.

16 · Cross Validation Added

What is Cross Validation? interview ★★

Cross Validation is a model evaluation technique that tests how well a model generalizes to unseen data. Instead of a single train/test split, it uses multiple splits to get a more reliable estimate of model performance and reduce overfitting risk.

Why not just train/test split? A single split is dependent on which data ends up in each set — lucky or unlucky split. Cross Validation averages over multiple splits to get a stable, unbiased estimate.

1. K-Fold Cross Validation interview ★★

Split data into K equal folds. Train on K-1 folds, test on 1 fold. Repeat K times, each fold acts as the test set once. Average the scores.

Key Properties

  • Every data point is used for both training and testing
  • Final score = average of all K scores
  • Typical K values: 5 or 10 (most common)
  • K=5 → 80% train, 20% test per fold
  • K=10 → 90% train, 10% test per fold
  • More reliable than single train/test split

Choosing K

K ValueTrade-offUse When
K = 5BalancedGeneral purpose ★
K = 10More accurate, slowerMedium datasets
K = n (LOOCV)Most accurate, very slowSmall datasets
K = 2 or 3Fast, less reliableLarge datasets

2. Stratified K-Fold interview ★

Same as K-Fold, but each fold maintains the same class proportion as the original dataset. Essential for imbalanced data.

Original dataset: 70% Class A, 30% Class B Regular K-Fold: Fold 1 test: 90% A, 10% B ← class imbalance per fold ❌ Fold 2 test: 60% A, 40% B Stratified K-Fold: Fold 1 test: 70% A, 30% B ← same ratio as original ✅ Fold 2 test: 70% A, 30% B ← consistent across folds ✅
Always use Stratified K-Fold for classification problems — especially when classes are imbalanced. sklearn: StratifiedKFold(n_splits=5)

3. LOOCV — Leave One Out Cross Validation interview ★

Special case of K-Fold where K = n (number of samples). Each iteration uses one sample as test and all remaining samples as training.

  • Train on n−1 samples, test on 1
  • Repeat n times (once per data point)
  • Final score = average of n scores
  • Uses almost all data for training each time
  • Most accurate estimate possible
  • Very slow — n model trainings
n = 100 samples Iteration 1: train on 99, test on sample 1 Iteration 2: train on 99, test on sample 2 ... Iteration 100: train on 99, test on sample 100 100 models trained → average = final score

Cross Validation Methods — Comparison

MethodTest sizeSpeedAccuracyBest For
K-Fold (K=5)20%FastGoodGeneral purpose ★
K-Fold (K=10)10%ModerateBetterMedium datasets
Stratified K-Fold20%FastGoodClassification + imbalanced ★
LOOCV1 sampleVery SlowBestSmall datasets
Repeated K-Fold20%SlowVery GoodStable estimate needed

Cross Validation for Model Selection

CV is used to: (1) Evaluate model performance reliably, (2) Select the best model among candidates, (3) Tune hyperparameters without overfitting to test data, (4) Detect overfitting — large gap between train CV score and test CV score.
17 · Hyperparameter Tuning Added

What is Hyperparameter Tuning? interview ★★

Hyperparameters are values set before training (not learned by the model). Tuning finds the optimal combination of hyperparameters that gives the best model performance.

Parameter vs Hyperparameter

ParameterHyperparameter
Learned byModel during trainingSet by you before training
ExamplesWeights (w), bias (b)Learning rate, n_estimators, max_depth, C, K
UpdatedVia gradient descentVia hyperparameter search

Common Hyperparameters by Algorithm

Linear/Logistic Reg: C (regularization) Decision Tree: max_depth, min_samples_split Random Forest: n_estimators, max_features XGBoost: learning_rate, n_estimators, max_depth, subsample, gamma SVM: C, kernel, gamma KNN: K (n_neighbors), metric

Method 1 — Grid Search CV interview ★

Try every possible combination of hyperparameter values from a predefined grid. Exhaustive but thorough.

How It Works

  • Define a grid of hyperparameter values to try
  • Train and evaluate model for every combination
  • Uses cross-validation to evaluate each combination
  • Pick combination with best CV score
param_grid = { 'max_depth': [3, 5, 7], 'n_estimators': [100, 200, 300], 'learning_rate': [0.01, 0.1] } Total combinations: 3 × 3 × 2 = 18 combinations Each with 5-fold CV → 18 × 5 = 90 model fits!

Pros & Cons

✅ Pros:
· Guaranteed to find best combination in the grid
· Simple to understand and implement
· Reproducible results
❌ Cons:
· Very slow — tries ALL combinations
· Computationally expensive for large grids
· Doesn't scale with many hyperparameters
· Grid must be defined manually
sklearn: GridSearchCV(estimator, param_grid, cv=5)

Method 2 — Random Search CV interview ★

Instead of trying all combinations, randomly sample a fixed number of hyperparameter combinations. Faster than Grid Search while often finding equally good results.

How It Works

  • Define a distribution or range for each hyperparameter
  • Randomly sample n_iter combinations
  • Evaluate each combination with cross-validation
  • Pick best combination found
param_dist = { 'max_depth': [3, 5, 7, 9, 11], 'n_estimators': range(50, 500), 'learning_rate': uniform(0.01, 0.3) } RandomizedSearchCV(n_iter=20) → tries only 20 random combos (not all!) → much faster, still effective ✓

Grid Search vs Random Search

Grid SearchRandom Search
SearchExhaustiveRandom sample
SpeedSlowFast ★
GuaranteeBest in gridMay miss best
Large gridsImpracticalScales well
Best forSmall param spaceLarge param space ★
Research finding: Random Search finds equally good hyperparameters as Grid Search in ~1/10 the time for large search spaces.

Method 3 — Bayesian Optimization interview ★★

Uses probabilistic model (surrogate function) to intelligently decide which hyperparameter combinations to try next — based on past results. Smarter than both Grid and Random Search.

Core idea: Instead of trying combinations blindly, Bayesian Optimization learns from previous trials and focuses on promising regions of the search space — like a smart student who learns from wrong answers.

How It Works — Step by Step

Step 1: Try a few random combinations (exploration) Step 2: Build a surrogate model (e.g. Gaussian Process) — models the relationship between hyperparams and performance Step 3: Use acquisition function to choose NEXT best combination to try Step 4: Evaluate that combination Step 5: Update surrogate model with result Step 6: Repeat Steps 3–5 until budget exhausted → Converges to optimal faster than Grid/Random

Pros & Cons

✅ Pros:
· Much more efficient than Grid/Random Search
· Learns from previous evaluations
· Finds good hyperparameters with fewer trials
· Handles continuous and discrete params
❌ Cons:
· More complex to implement
· Sequential — can't parallelize easily
· Surrogate model has its own overhead
Libraries: Optuna, Hyperopt, Scikit-Optimize (skopt), BayesSearchCV

All Three Methods — Full Comparison

Grid SearchRandom SearchBayesian Optimization
Strategy Try all combos Try random combos Learn & focus on promising regions
Speed Slowest Fast Most efficient ★
Trials needed All combinations n_iter (e.g. 20–50) Few (e.g. 10–30)
Uses past results ❌ No ❌ No ✅ Yes

sklearn GridSearchCV RandomizedSearchCV Optuna / skopt

When to Use Which?

Small param grid (≤ 20 combos)? → Grid Search — exhaustive, guaranteed best in grid Large param grid (100s of combos)? → Random Search — fast, good enough Expensive model (deep learning, XGBoost) + limited budget? → Bayesian Optimization (Optuna) — most efficient Production / competition ML? → Bayesian Optimization + cross-validation
18 · Quick Revision Cheatsheet

EDA Steps

  • Profile — rows, cols, types, missing, duplicates
  • Statistical — mean, median, std, variance, tests
  • Graph — histogram, boxplot, scatter, heatmap, KDE

Missing Values

  • MCAR — random, use CCA or imputation
  • MAR — depends on other vars, use imputation
  • NMAR — depends on itself, domain knowledge needed
  • Numerical: mean (normal) / median (skewed) / KNN
  • Categorical: mode / "Missing" category

Scaling

  • Standardization — mean=0, std=1, (−∞, +∞)
  • Normalization — range [0,1], CNN inputs
  • Tree-based algos: no scaling needed
  • Distance-based algos: always scale

Encoding

  • Label — ordinal data (order matters)
  • One-Hot — nominal, few categories
  • Binary — nominal, many categories

Feature Selection Methods

  • Filter — Variance, Correlation, ANOVA, Chi-Square, MI
  • Wrapper — Forward, Backward, Exhaustive, RFE
  • Embedded — Lasso (L1), Ridge (L2), Tree importance

Imbalanced Data

  • Undersampling — remove majority class samples
  • Oversampling — duplicate minority samples
  • SMOTE — synthetic interpolated samples
  • Use F1, AUC — not accuracy

PCA vs LDA

  • PCA — unsupervised, max variance, no labels
  • LDA — supervised, max class separation, uses labels
  • Keep PCs until cumulative variance ≥ 90%

Data Drift New

  • Covariate Shift — P(X) changes, input features shift
  • Concept Drift — P(Y|X) changes, relationship changes
  • Prior Shift — P(Y) changes, class proportions shift
  • Detect: PSI, KS Test, Chi-Square, Jensen-Shannon
  • PSI <0.1 ✅ | 0.1–0.2 ⚠️ | >0.2 ❌ retrain
  • Fix: Retrain, sliding window, online learning
  • Tools: Evidently AI, WhyLogs, Alibi Detect

Cross Validation New

  • K-Fold — split into K folds, rotate test fold, avg scores
  • Stratified K-Fold — maintains class ratio per fold ★
  • LOOCV — K=n, one sample test, most accurate but slow
  • K=5 or 10 is standard
  • Use Stratified for classification + imbalanced data
  • Use LOOCV for small datasets

Hyperparameter Tuning New

  • Grid Search — all combos, slow, guaranteed best in grid
  • Random Search — random combos, fast, large grids
  • Bayesian Optimization — learns from past, most efficient
  • Small grid → Grid Search | Large grid → Random / Bayesian
  • Expensive model → Optuna (Bayesian) ★
  • Always use CV inside tuning to avoid overfitting
