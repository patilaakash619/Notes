01 · Machine Learning Basics

What is Machine Learning?

Subfield of AI that focuses on developing algorithms and models that enable computers to learn from data and improve performance without being explicitly programmed.

Core idea → machines automatically learn patterns, make predictions, and take decisions based on input data.

1. Supervised Learning

Learning from labeled data (data with answers)

  • Model trained where output is already known
  • Learns: Input → Correct Output

Examples

  • Classification — predicting labels/categories
  • Regression — predicting numbers (continuous)

2. Unsupervised Learning

Finding patterns in unlabeled data — learning without answers

  • Clustering — grouping similar data points
  • Dimensionality Reduction — reducing features, retaining info
  • Association — discovering rules among variables

3. Reinforcement Learning

An agent learns by interacting with environment, improving actions based on rewards and penalties.

Agent (Model) Environment Action Rewards / Penalties State

04 · Linear Regression

Simple Linear Regression

Foundation technique for predicting a continuous target variable based on one or more input features.

y = β₀ + β₁x
SymbolMeaning
yDependent variable (target) — what you're predicting
xIndependent variable (feature) — your input
β₁Slope — how much y changes when x increases by 1
β₀Intercept — value of y when x = 0
Example: Score = 40 + 5 × (Hours Studied) → studying 6 hrs → score = 70

Best Fit Line — OLS

Algorithm minimizes Sum of Squared Residuals — called Ordinary Least Squares (OLS)

  • Residual = (actual y) − (predicted y)
  • Squaring ensures −ve & +ve errors don't cancel
  • Larger errors are penalized more

Multiple Linear Regression

y = β₀ + β₁x₁ + β₂x₂ + β₃x₃ + ...
Same idea — just more variables. Example: predicting house price using size (x₁), bedrooms (x₂), distance from city (x₃)
05 · Loss & Cost Functions
TermApplies toMeaning
Loss Function1 data pointHow wrong is prediction for a single example
Cost FunctionEntire datasetAverage of all losses across all data points

1. MSE — Mean Squared Error

(1/n) × Σ(actual − predicted)²

Behaviour

  • Squares error → big mistakes punished heavily
  • Smooth curve → easy for Gradient Descent
  • Sensitive to outliers

Use When

  • Want to penalize large errors heavily
  • Data has few/no outliers
  • Default choice for regression

2. MAE — Mean Absolute Error

(1/n) × Σ|actual − predicted|

Behaviour

  • Treats all errors equally (no squaring)
  • Robust to outliers
  • Not smooth at zero → slightly harder to optimize

Use When

  • Data has outliers you don't want to over-penalize
  • Want equal treatment of all errors
  • Median-based prediction is acceptable

3. Huber Loss — Best of Both

Small error (< δ)

Acts like MSE — smooth, penalizes moderately

Large error (> δ)

Acts like MAE — robust, doesn't over-punish

Use when: Real-world messy data with some outliers — smooth optimization AND outlier robustness
06 · Gradient Descent

What is Gradient Descent?

The algorithm that minimizes the cost function. It's the engine that drives learning.

Steps

Step 1: Start with random weights Step 2: Calculate cost (how wrong is model?) Step 3: Calculate gradient (which direction is downhill?) Step 4: Update weights → move downhill Step 5: Repeat until cost is minimum ✓

Update Formula

w = w − α × (∂Cost/∂w)
SymbolMeaning
wModel weight (coefficient)
αLearning rate (step size)
∂Cost/∂wGradient (direction to move)

Types of Gradient Descent

TypeData per stepSpeedUse when
Batch GDAll data🐢 SlowSmall dataset
SGD1 sample⚡ FastVery large datasets
Mini-Batch GD ★ Default32–128 samples✅ BalancedAlmost always

Learning Rate

α valueProblem
Too largeOvershoots minimum — never converges
Too smallTakes forever to train
Just rightReaches minimum efficiently ✅

Adam Optimizer interview ★

Adam = Momentum + Adaptive Learning Rate. Automatically adjusts learning rate per parameter. Default choice for deep learning — faster and more reliable than basic GD.
07 · Overfitting & Underfitting
ProblemTrain ErrorTest ErrorCause

Fix Underfitting

  • Add more features
  • Add polynomial features (for curves)
  • Reduce regularization (lower λ)
  • Use more complex model
  • Train longer (Neural Nets)
  • Clean noisy data

Fix Overfitting

  • Get more training data
  • Remove useless features
  • Regularization — Lasso / Ridge
  • Cross-validation (K-Fold)
  • Simpler model
  • Early stopping (Neural Nets)
08 · Regularization (for Overfitting)
⚠️ Regularization is ONLY for Overfitting. Never use for underfitting — it makes it worse.

L1 — Lasso

Penalty = sum of |coefficients|

  • Pushes some coefficients to exactly zero
  • Removes useless features automatically
  • Use when: many features, most likely useless

L2 — Ridge

Penalty = sum of coefficients²

  • Shrinks all coefficients but none reach zero
  • Keeps all features, just smaller
  • Use when: all features genuinely matter

ElasticNet = L1 + L2 Best of both

Use when unsure, or features are correlated, or large dataset.

Lambda (λ) — Strength Dial

λ = 0 No penalty → original model λ small Mild shrink λ right Good for overfitting ✓ λ large Over-simplified → underfit!

Find best λ using Cross-Validation.

10 · R² vs Adjusted R²

R²

"What % of the output does my model explain?"

R² = 1 − SS_res / SS_tot
ValueMeaning
1.0Perfect model
0.85Good
0.5Mediocre
0.0Useless
NegativeWorse than average
⚠️ Problem: Always increases when you add features — even useless ones!

Adjusted R²

Same as R², but penalizes useless features

1 − [(1−R²)(n−1) / (n−k−1)]
ActionResult
Add useful featureGoes up ↑
Add useless featureGoes down ↓

When to Use Which

  • Same number of features → R²
  • Different number of features → Adjusted R²
  • Adding/removing features → Adjusted R²
11 · Linear Regression Assumptions
#AssumptionSimple MeaningCheck
1 Linear Relationship X and Y must have a straight-line relationship Scatter plot
2 No Multicollinearity interview ★ Input features should not be highly correlated with each other.
VIF (Variance Inflation Factor) — measures how much one feature is explained by other features.
VIF < 5 ✅ Acceptable  |  VIF > 10 ❌ High multicollinearity — remove that feature
Correlation matrix, VIF score
3 Homoscedasticity Errors should be equally spread at all values of X (no fan shape) Residual plot
4 Normal Distribution of Errors Residuals should follow a bell curve Histogram of residuals
5 No Autocorrelation Each data point's error should be independent of others (common in time-series) Durbin-Watson test ≈ 2 ✅
If any assumption is violated → model predictions become unreliable. Always check assumptions before trusting your model.
12 · Quick Revision Cheatsheet

Regression Metrics

  • MAE — avg absolute error, robust
  • MSE — penalizes large errors
  • RMSE — √MSE, same unit as y
  • R² — % variation explained
  • Adj R² — honest R² for multi-features

Loss Functions

  • MSE — clean data, penalize big errors
  • MAE — noisy data, equal treatment
  • Huber — mix of both, real-world
  • Binary Cross-Entropy — 2-class problems
  • Categorical CE — multi-class problems

Regularization

  • L1 Lasso — kills useless features (→ 0)
  • L2 Ridge — shrinks all features
  • ElasticNet — L1 + L2 combined
  • Only for Overfitting ❌ not underfitting

Gradient Descent

  • Batch — all data, slow
  • SGD — 1 sample, fast, noisy
  • Mini-Batch — balanced ★ default
  • Adam — Momentum + Adaptive LR, best for DL
  • VIF < 5 ✅  |  VIF > 10 ❌ multicollinearity
