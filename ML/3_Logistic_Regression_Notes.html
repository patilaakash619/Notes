01 · What is Logistic Regression?

Definition

A classification algorithm used to predict the probability of a categorical outcome (class). Despite the name "regression", it is used for classification — not regression.

Key idea: Instead of predicting a number like Linear Regression, Logistic Regression predicts a probability between 0 and 1, then converts it into a class label.

Linear Regression vs Logistic Regression

OutputAny number0 to 1 (prob)
TaskRegressionClassification
LineStraight lineS-shaped curve
LossMSECross-Entropy

Real World Examples

  • Email → Spam or Not Spam
  • Tumor → Malignant or Benign
  • Loan → Approved or Rejected
  • Student → Pass or Fail
  • Transaction → Fraud or Legit
02 · Sigmoid Function

The Core Function interview ★

Logistic Regression uses the Sigmoid function to squash any value into a probability between 0 and 1.

σ(z) = 1 / (1 + e⁻ᶻ)    where z = β₀ + β₁x₁ + β₂x₂ + ...

How it works

  • Input z = any real number (−∞ to +∞)
  • Output σ(z) = always between 0 and 1
  • z → +∞ means σ → 1
  • z → −∞ means σ → 0
  • z = 0 means σ = 0.5 (decision boundary)

Decision Rule

if σ(z) ≥ 0.5 → Predict Class 1 if σ(z) < 0.5 → Predict Class 0
The threshold (0.5) can be changed depending on the problem. E.g. for cancer detection, lower it to 0.3 to catch more cases.
03 · Logistic Regression Equation

The Math

z = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ
P(y=1) = σ(z) = 1 / (1 + e⁻ᶻ)
SymbolMeaning
zLinear combination of inputs (same as linear regression)
β₀Intercept (bias)
β₁...βₙCoefficients (weights) learned during training
P(y=1)Probability that the output is class 1
eEuler's number ≈ 2.718

Log Odds (Logit) interview ★

Logistic Regression actually models the log of odds, not probability directly.

log(P / 1−P) = β₀ + β₁x₁ + ...
  • Odds = P(event) / P(no event)
  • Log odds = logit transformation → makes output linear
  • This is why it's called Logistic Regression

Binary

  • Yes / No
  • 0 / 1

  • Red / Blue / Green
  • News: Sports / Tech / Politics
Uses Softmax function

Ordinal

Ordered Classes

Output has ordered categories

  • Low / Medium / High
  • Poor / Good / Excellent

05 · Loss Function — Binary Cross-Entropy

Log Loss / Binary Cross-Entropy interview ★

Logistic Regression does NOT use MSE. It uses Binary Cross-Entropy as its loss function.

Loss = −[y·log(p) + (1−y)·log(1−p)]
SymbolMeaning
yActual label (0 or 1)
pPredicted probability (0 to 1)
logNatural logarithm

Why NOT MSE?

MSE with sigmoid produces a non-convex loss curve (many local minima) — Gradient Descent struggles. Cross-Entropy gives a convex curve — single minimum, easy to optimize.

How it penalizes

Actual = 1 (Spam) Predicted = 0.9 → small loss ✓ confident and correct Predicted = 0.5 → medium loss ~ uncertain Predicted = 0.1 → HUGE loss ✗ confident and WRONG ← heavily punished Actual = 0 (Not Spam) Predicted = 0.1 → small loss ✓ Predicted = 0.9 → HUGE loss ✗ confident wrong prediction

Cost Function (average over all samples)

J(β) = −(1/n) × Σ [yᵢ·log(pᵢ) + (1−yᵢ)·log(1−pᵢ)]

The 4 Key Metrics interview ★

Accuracy

(TP + TN) / (TP + TN + FP + FN)

% of total correct predictions

⚠️ Misleading on imbalanced data. 99% accuracy on 1% fraud dataset means nothing if it predicts everything as "not fraud".

Precision interview ★

TP / (TP + FP)

Of all predicted positives, how many were actually positive?

Use when: FP is costly — e.g. spam filter (don't mark legit emails as spam)

Recall (Sensitivity) interview ★

TP / (TP + FN)

Of all actual positives, how many did we catch?

Use when: FN is costly — e.g. cancer detection (don't miss sick patients!)

F1 Score interview ★

2 × (Precision × Recall) / (Precision + Recall)

Harmonic mean of Precision and Recall

Use when: You need balance between Precision and Recall — best for imbalanced datasets

Precision vs Recall Tradeoff interview ★

You cannot maximize both at the same time. Increasing one usually decreases the other.

High Precision needed

  • Spam detection (don't block good emails)
  • Legal document classification
  • Product recommendations

→ Raise threshold (e.g. 0.7)

High Recall needed

  • Cancer / disease detection
  • Fraud detection
  • Defect detection in manufacturing

→ Lower threshold (e.g. 0.3)

09 · Multi-class Classification

Handling More Than 2 Classes

One vs Rest (OvR)

  • Train one model per class
  • Cat vs (Dog+Bird), Dog vs (Cat+Bird)...
  • Pick class with highest probability
Simple, works well in practice

Softmax Function

P(class k) = eᶻᵏ / Σ eᶻʲ
  • Outputs probabilities for all classes
  • All probabilities sum to 1
  • Used with Categorical Cross-Entropy
Softmax example: [Cat: 0.70, Dog: 0.20, Bird: 0.10] → All sum to 1 → Predict Cat
10 · Regularization in Logistic Regression

Same as Linear Regression — add penalty to prevent overfitting.

L1 — Lasso sparse

  • Pushes some weights to exactly zero
  • Automatic feature selection
  • Use when many features are irrelevant

L2 — Ridge smooth

  • Shrinks all weights but none to zero
  • Handles correlated features well
  • Use when all features matter
In sklearn, the regularization parameter is C = 1/λ. So smaller C = stronger regularization (opposite of λ).
⚠️ Regularization is only for overfitting. Never use for underfitting.

13 · Quick Revision Cheatsheet

Core Concepts

  • Logistic Regression = classification, not regression
  • Sigmoid = squashes output to 0–1
  • Threshold = 0.5 by default (adjustable)
  • Log Loss = loss function (Binary Cross-Entropy)
  • Decision Boundary = where z = 0, σ = 0.5
  • Softmax = multi-class version of sigmoid

Types

  • Binary — 2 classes → Sigmoid
  • Multinomial — 3+ classes → Softmax
  • Ordinal — ordered classes

Evaluation Metrics

  • Accuracy — overall correct / total
  • Precision — correct positives / predicted positives
  • Recall — correct positives / actual positives
  • F1 — balance of Precision + Recall
  • AUC-ROC — area under ROC curve (1 = perfect)

When to use what

  • FP costly → maximize Precision → raise threshold
  • FN costly → maximize Recall → lower threshold
  • Imbalanced data → use F1 or AUC, not Accuracy
  • VIF < 5 ✅  |  VIF > 10 ❌ multicollinearity
  • C in sklearn = 1/λ → smaller C = more regularization
